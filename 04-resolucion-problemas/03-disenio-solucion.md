# Diseño de la Solución

## ¿Qué es el diseño de la solución?

El diseño de la solución consiste en planificar los pasos necesarios para resolver un problema antes de representarlo mediante pseudocódigo o diagramas de flujo.

Durante esta etapa se define la lógica que seguirá el algoritmo.

---

# Importancia

El diseño de la solución permite:

* Organizar la lógica del algoritmo.
* Identificar recursos necesarios.
* Reducir errores de implementación.
* Facilitar la construcción del pseudocódigo.
* Mejorar la calidad de las soluciones.

---

# Relación con la resolución de problemas

El diseño de la solución se realiza después del análisis del problema.

## Flujo de trabajo

```text
Problema
    ↓
Análisis del problema
    ↓
Diseño de la solución
    ↓
Pseudocódigo
    ↓
Diagrama de flujo
    ↓
Prueba de escritorio
    ↓
Implementación
    ↓
Pruebas y corrección
```

---

# Componentes del diseño

## Secuencia lógica

Describe los pasos necesarios para resolver el problema.

### Ejemplo

1. Leer datos.
2. Procesar información.
3. Mostrar resultados.

---

## Variables utilizadas

Son los datos que almacenará el algoritmo durante su ejecución.

### Ejemplo

| Variable | Descripción              |
| -------- | ------------------------ |
| edad     | Edad del usuario         |
| salario  | Salario calculado        |
| contador | Contador de repeticiones |

---

## Operadores utilizados

Permiten realizar operaciones.

### Operadores aritméticos

| Operador | Uso            |
| -------- | -------------- |
| +        | Suma           |
| -        | Resta          |
| *        | Multiplicación |
| /        | División       |
| %        | Residuo        |

### Operadores relacionales

| Operador | Uso           |
| -------- | ------------- |
| ==       | Igual         |
| !=       | Diferente     |
| >        | Mayor         |
| <        | Menor         |
| >=       | Mayor o igual |
| <=       | Menor o igual |

### Operadores lógicos

| Operador | Uso      |
| -------- | -------- |
| AND      | Y        |
| OR       | O        |
| NOT      | Negación |

---

## Estructuras utilizadas

Permiten controlar el flujo del algoritmo.

### Secuencial

Las instrucciones se ejecutan una después de otra.

### Condicional

Permite tomar decisiones.

Ejemplo:

```text
Si edad >= 18
```

### Repetitiva

Permite repetir acciones.

Ejemplo:

```text
Para i ← 1 Hasta 10
```

---

## Fórmulas utilizadas

Son expresiones matemáticas necesarias para resolver el problema.

### Ejemplo

```text
area = base * altura
```

```text
anio_nacimiento = 2026 - edad
```

---

# Ejemplo práctico

## Problema

Calcular el área de un rectángulo.

### Secuencia lógica

1. Leer base.
2. Leer altura.
3. Calcular área.
4. Mostrar resultado.

### Variables utilizadas

base, altura, area

### Operadores utilizados

*

### Estructuras utilizadas

Secuencial

### Fórmulas utilizadas

```text
area = base * altura
```

---

# Recomendaciones

* Diseñar antes de programar.
* Utilizar nombres descriptivos.
* Identificar estructuras necesarias.
* Verificar la lógica antes de continuar.

---

# Errores comunes

| Error                          | Descripción                         |
| ------------------------------ | ----------------------------------- |
| Omitir pasos                   | Produce soluciones incompletas.     |
| No identificar variables       | Genera errores posteriores.         |
| Elegir estructuras incorrectas | Complica la solución.               |
| No definir fórmulas            | Impide realizar cálculos correctos. |

---

# Conclusión

El diseño de la solución transforma el análisis del problema en una estrategia organizada que posteriormente podrá representarse mediante pseudocódigo y diagramas de flujo.
