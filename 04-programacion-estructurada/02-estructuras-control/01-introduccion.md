# Introducción a las Estructuras de Control

## ¿Qué son las estructuras de control?

Las **estructuras de control** son mecanismos que permiten controlar el flujo de ejecución de un algoritmo o programa.

Gracias a ellas es posible determinar el orden en que se ejecutan las instrucciones, tomar decisiones y repetir acciones cuando sea necesario.

Sin estructuras de control, los algoritmos únicamente podrían ejecutar instrucciones de forma lineal.

---

# Importancia

Las estructuras de control permiten:

- Organizar la lógica de una solución.
- Tomar decisiones según determinadas condiciones.
- Repetir procesos de forma automática.
- Resolver problemas más complejos.
- Construir algoritmos más flexibles y eficientes.

Son uno de los pilares fundamentales de la programación estructurada.

---

# Flujo de ejecución

Por defecto, las instrucciones de un algoritmo se ejecutan en el orden en que fueron escritas.

```text
Instrucción 1
      ↓
Instrucción 2
      ↓
Instrucción 3
      ↓
Instrucción 4
```

Este comportamiento se conoce como **flujo secuencial**.

Sin embargo, muchos problemas requieren modificar ese flujo para tomar decisiones o repetir acciones, lo que hace necesario el uso de estructuras de control.

---

# Las tres estructuras fundamentales

La programación estructurada se basa en tres estructuras fundamentales.

## 1. Secuencia

Las instrucciones se ejecutan una después de otra siguiendo un orden determinado.

Ejemplo de uso:

- Leer datos.
- Realizar cálculos.
- Mostrar resultados.

---

## 2. Selección

Permite elegir entre diferentes caminos de ejecución según se cumpla o no una condición.

Ejemplo de uso:

- Determinar si una persona es mayor de edad.
- Verificar si un número es positivo o negativo.
- Validar credenciales de acceso.

---

## 3. Repetición

Permite ejecutar un conjunto de instrucciones varias veces.

Ejemplo de uso:

- Mostrar una serie de números.
- Procesar varios registros.
- Repetir un menú de opciones.

---

# Clasificación de las estructuras de control

Las estructuras de control pueden clasificarse en tres grupos principales.

| Tipo | Función |
|--------|----------|
| Secuenciales | Ejecutan instrucciones en orden. |
| Condicionales | Permiten tomar decisiones. |
| Repetitivas | Permiten repetir instrucciones. |

---

# Estructuras secuenciales

Las instrucciones se ejecutan una después de otra siguiendo el orden establecido.

Son las estructuras más simples y constituyen la base de cualquier algoritmo.

Este tema será estudiado en el siguiente apartado.

---

# Estructuras condicionales

Permiten ejecutar diferentes acciones dependiendo del resultado de una condición.

Algunas estructuras condicionales son:

- if simple
- if else
- if anidado
- switch

Estas estructuras serán estudiadas en los siguientes temas.

---

# Estructuras repetitivas

Permiten ejecutar un bloque de instrucciones varias veces.

Algunas estructuras repetitivas son:

- while
- do while
- for

Estas estructuras serán estudiadas posteriormente.

Las estructuras de control se utilizan en prácticamente cualquier programa:

- Sistemas bancarios.
- Aplicaciones móviles.
- Videojuegos.
- Sistemas académicos.
- Sistemas empresariales.

# Errores comunes

| Error | Descripción |
|---------|------------|
| Utilizar una estructura incorrecta | Puede complicar la solución. |
| Plantear condiciones erróneas | Produce resultados inesperados. |
| Crear ciclos infinitos | Impide finalizar la ejecución. |
| Omitir casos posibles | Genera comportamientos incorrectos. |

---

# Conclusión

Las estructuras de control permiten dirigir el flujo de ejecución de un algoritmo. Gracias a ellas es posible construir soluciones capaces de tomar decisiones, repetir acciones y resolver problemas de forma organizada y eficiente.

Toda la programación estructurada se fundamenta en tres estructuras básicas: secuencia, selección y repetición.
